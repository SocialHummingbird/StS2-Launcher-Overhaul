Current release decision: not release-ready. Static implementation/docs/CI guardrails are in place, but ARM64 evidence is still missing for public/default regression, beta branch download/startup routing, cache cleanup, inaccessible/private/password branch behavior, Pull-after-switch/current-backup safety, Autofill provider behavior, artifact hygiene review, and save compatibility. The release-readiness tracker is the source of truth for the current signoff contract.

Current implementation status:

- The launcher persists the selected Steam branch in `game_branch`.
- The launcher exposes game version selection as a discovery-led dropdown. `public` remains available by default, account-visible branches are added from Steam app-info after refresh, and previously saved custom branch values are preserved as selectable options for compatibility/retry diagnostics without reintroducing manual text entry.
- During download/update app-info resolution, the downloader logs account-visible Steam branch availability, Windows depot manifest counts per visible branch, and selected-branch visibility/downloadability evidence for failed non-public downloads. It also writes `last_steam_branch_availability.txt` so diagnostics can report the latest branch availability evidence after the live log has scrolled away. Failed download/update UI status should compact that marker into a user-facing branch availability diagnosis without requiring adb/logcat.
- The default/public branch uses the legacy `files/game` and `files/download_state` paths.
- Non-public branches use side-by-side caches under `files/game_versions/<safe-branch>/game` and `files/game_versions/<safe-branch>/download_state`, with diagnostics naming the active install slot kind and slot directory.
- Download, update check, redownload, native startup, managed startup fallback, diagnostics, and inactive-cache cleanup are branch-aware.
- Completed downloads write `steam_branch.txt` into the selected game directory so branch ownership and install slot can be audited from the installed cache.
- Branch switching shows a warning and enables local backups before applying the selected branch.

Current limitations:

- The UI is dropdown-first and can populate the dropdown from the latest Steam app-info branch availability marker after download/update app-info has been captured. It now exposes a non-mutating `REFRESH GAME VERSIONS` action that fetches Steam app-info branch metadata before download and refreshes the dropdown without downloading or deleting game files. Dropdown labels should remain concise for phone screens, while selector helper text should show the active install slot and selected-version availability details. Diagnostics should show `Steam branch selector mode: Steam branch dropdown`, selected branch preference key/source/selection kind, `Steam branch discovery supported: true`, Steam branch catalog source, Steam branch dropdown options, and Steam branch dropdown option metadata including source, metadata visibility, Windows manifest count, password flag, and build ID when available. Download/update logs and `last_steam_branch_availability.txt` should still surface account-visible Steam branch availability from app info.
- Steam login hardening uses Android/Samsung/password-manager Autofill as the credential provider model. The launcher must not store or log Steam passwords for Autofill. Current Godot login fields carry username/password metadata, and Android builds expose a one-shot native Autofill dialog with real username/password `EditText` controls. Filled native values are held in memory only until consumed by the existing login flow, cancelled, the Android activity stops/destroys, or expired after 60 seconds; the Godot password field is cleared immediately after login request capture. Native Autofill behavior still needs ARM64 device validation with Samsung/Android/password-manager providers.
- Steam beta password entry is not implemented.
- If the selected non-public branch has no accessible manifests, download/update checks should fail with a branch-specific message naming missing/private/password-protected beta possibilities.
- Save-format compatibility between branches is unknown.
- ARM64 hardware remains the proof target for Steam login, depot download, branch switching, cloud saves, and game launch.
