Current release decision: not release-ready. Static implementation/docs/CI guardrails are in place, but ARM64 evidence is still missing for public/default regression, beta branch download/startup routing, cache cleanup, inaccessible/private/password branch behavior, Pull-after-switch/current-backup safety, and save compatibility.

Current implementation status:

- The launcher persists the selected Steam branch in `game_branch`.
- The launcher currently exposes a `public/beta` toggle and does not support arbitrary Steam branch discovery.
- The default/public branch uses the legacy `files/game` and `files/download_state` paths.
- Non-public branches use side-by-side caches under `files/game_versions/<safe-branch>/game` and `files/game_versions/<safe-branch>/download_state`, with diagnostics naming the active install slot kind and slot directory.
- Download, update check, redownload, native startup, managed startup fallback, diagnostics, and inactive-cache cleanup are branch-aware.
- Completed downloads write `steam_branch.txt` into the selected game directory so branch ownership and install slot can be audited from the installed cache.
- Branch switching shows a warning and enables local backups before applying the selected branch.

Current limitations:

- The UI currently exposes a default/public versus `beta` toggle, not arbitrary branch discovery. Selector helper text should show the active install slot. Diagnostics should show `Steam branch selector mode: public/beta toggle` and `Steam branch discovery supported: false`.
- Steam beta password entry is not implemented.
- If the selected non-public branch has no accessible manifests, download/update checks should fail with a branch-specific message naming missing/private/password-protected beta possibilities.
- Save-format compatibility between branches is unknown.
- ARM64 hardware remains the proof target for Steam login, depot download, branch switching, cloud saves, and game launch.
